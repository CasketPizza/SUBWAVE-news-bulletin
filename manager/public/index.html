Hourly News Bulletin

A scheduled radio news programme with an intro, clearly separated stories over a bed, and an outro—using SUB/WAVE’s LLM with a configurable presenter and voice.

This companion does not patch or replace any SUB/WAVE application files. It is mounted at /news-bulletin/ through a reversible Caddy overlay.

Schedule

A portable companion cannot receive an “hourly announcement finished” event from SUB/WAVE, so the default uses :01. In programme-item mode, the current song is ended and the bulletin is placed ahead of pending songs.

Newsroom & presenter

Control which SUB/WAVE personality writes the bulletin, which voice reads it, and the instructions given to the LLM.

The list is populated from voices SUB/WAVE already knows about. A voice name or ID can also be typed manually for engines that accept one.

Accuracy safeguards remain enforced even when these instructions are customised: the bulletin cannot invent facts, merge unrelated reports, or turn allegations and uncertainty into certainty. Audience interests are only a soft tie-breaker and cannot displace important news.

News sources

RSS 2.0 and Atom feeds are accepted. Sources are mixed round-robin.

Complete report playback

The story count controls how many items are selected. The generated report is never shortened to a time target or silently truncated by the TTS preview limit.

Audio package

Choosing a file uploads and selects it immediately. The original file name is shown once conversion finishes.

When enabled, SUB/WAVE skips the current song once, then airs one verified intro/news/outro file from beginning to end. Pending songs stay outside the queue for the full package, and a protected silent tail absorbs the outgoing crossfade so the next song cannot consume the end of the report. Disable this only to use ordinary foreground-voice ducking instead.
